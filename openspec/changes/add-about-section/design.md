## Context

当前个人品牌站已有 Hero Section（全屏展示姓名和职业）、Project Section（项目展示）和 Contact Section（联系方式）。需要在 Hero 和 Projects 之间添加一个"关于我"区域，提供更详细的个人背景介绍，填补"我是谁"和"我做了什么"之间的信息断层。

## Goals / Non-Goals

**Goals:**
- 创建 AboutSection 组件，采用左右分栏布局（照片 + 简介）
- 个人照片使用占位图或用户提供的图片，支持懒加载
- 个人简介分为三段：背景经历、专业理念、职业追求
- 支持亮/暗主题适配
- 响应式布局：桌面端左右排列，移动端上下堆叠

**Non-Goals:**
- 联系表单
- 动态编辑功能
- 社交媒体图标链接
- 时间线履历
- 技能图表/进度条

## Decisions

### 1. 布局结构
**决策**: 使用 Flexbox 左右分栏，移动端垂直堆叠  
**理由**:
- `flex-col md:flex-row` 实现响应式切换
- 桌面端照片占 40%，文字占 60%，黄金比例
- 移动端照片在上，文字在下，阅读顺序自然

### 2. 照片处理
**决策**: 使用圆角矩形 + 轻微阴影，占位图使用 Unsplash  
**理由**:
- `rounded-2xl` 圆角符合现代设计趋势
- `aspect-[3/4]` 竖版照片比例更适合人像
- `object-cover` 保持图片比例
- 懒加载优化性能
- 添加边框增加层次感

### 3. 文字排版
**决策**: 三段式结构，使用合适的字体层级  
**理由**:
- 标题使用 `text-3xl font-bold`
- 每段简介使用 `text-base leading-relaxed` 增加可读性
- 段落间距 `space-y-4` 清晰分隔

### 4. 主题适配
**决策**: 亮色使用白色背景，暗色使用深灰背景  
**理由**:
- 与现有 section 保持一致的背景策略
- 卡片/文字颜色通过 `dark:` 变体切换
- 照片边框颜色适配主题

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| 用户照片未提供 | 使用高质量 Unsplash 占位图，用户可自行替换 |
| 文字内容过长 | 限制每段 100-150 字，保持简洁 |
| 品牌标签过于突兀 | 使用柔和的颜色和适当的间距融入设计 |
| 移动端照片过大 | 限制最大宽度 `max-w-sm mx-auto` |
